    **ApprovalService (core state machine per RESEARCH.md Pattern 3):**
    - @Service, @Transactional on all write methods
    
    - State machine validation: use ApprovalAction enum from RESEARCH.md lines 534-558
      - Transition map: DRAFT -> SUBMIT -> PENDING_DEPT_REVIEW
      - PENDING_DEPT_REVIEW -> PASS_DEPT -> PENDING_ADMIN_ARCHIVE
      - PENDING_DEPT_REVIEW -> REJECT_DEPT -> REJECTED  
      - PENDING_ADMIN_ARCHIVE -> PASS_ADMIN -> ARCHIVED
      - PENDING_ADMIN_ARCHIVE -> REJECT_ADMIN -> REJECTED
      - PENDING_DEPT_REVIEW -> WITHDRAW -> WITHDRAWN (submitter only)
      - PENDING_ADMIN_ARCHIVE -> WITHDRAW -> WITHDRAWN (submitter only)
      - REJECTED -> RESUBMIT -> PENDING_DEPT_REVIEW
      - ARCHIVED -> INVALIDATE -> INVALIDATED (done in Plan 01-05)
    
    - submit(Long achievementId, String type, Long userId):
      - Load achievement by type + id
      - Validate current status == DRAFT
      - Check duplicate (call DuplicateCheckService)
      - Transition status to PENDING_DEPT_REVIEW
      - Create ApprovalRecord with action=SUBMIT
      - Call AuditLogService
      - Notify department secretaries (call NotificationService)
    
    - approve(Long achievementId, String type, Long approverId, String archiveNo):
      - Load achievement, validate status
      - If PENDING_DEPT_REVIEW: validate approver is dept secretary of same dept (D-31). Transition to PENDING_ADMIN_ARCHIVE. Notify admin.
      - If PENDING_ADMIN_ARCHIVE: validate approver is admin. Transition to ARCHIVED. Set archiveNo (D-33) using format PER-YEAR-#### or PAT-YEAR-#### or CTR-YEAR-####. Notify submitter "您的成果已归档，编号：{archiveNo}".
      - Create ApprovalRecord for each step
      - Use SELECT ... FOR UPDATE on achievement row for concurrency safety (Pitfall 1)
    
    - reject(Long achievementId, String type, Long approverId, String reason):
      - Load achievement, validate status (PENDING_DEPT_REVIEW or PENDING_ADMIN_ARCHIVE)
      - Transition to REJECTED
      - Create ApprovalRecord with action=REJECT_DEPT/REJECT_ADMIN
      - Notify submitter "您的成果被退回: {reason}"
    
    - withdraw(Long achievementId, String type, Long userId):
      - Load achievement, validate status (PENDING_DEPT_REVIEW or PENDING_ADMIN_ARCHIVE)
      - Validate userId == created_by of achievement (submitter only, per D-29)
      - Transition to WITHDRAWN
      - Create ApprovalRecord with action=WITHDRAW
    
    - getPendingApprovals(Long userId, String type, String dateRange, int page, int size):
      - For dept secretary: query achievements where status=PENDING_DEPT_REVIEW AND dept_id = user.deptId
      - For admin: query achievements where status=PENDING_ADMIN_ARCHIVE
      - Return paginated results with achievement title, type, submitter, department, submit time
    
    - getApprovalHistory(String achievementType, Long achievementId):
      - Return List<ApprovalRecordVO> ordered by created_time ASC
    
    - getDetail(String achievementType, Long achievementId):
      - Return full achievement fields (read-only) for display in approval detail page
    
    - Dynamic entity update helper methods:
      - updateStatus(type, id, newStatus): dispatches to PaperMapper/PatentMapper/CopyrightMapper updateById with status param
      - getEntity(type, id): dispatches to correct mapper
