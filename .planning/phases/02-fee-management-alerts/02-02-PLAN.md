<objective>
**Plan 02-02: Payment Plan Engine — Single + Recurring Annual Fee Generation**

This plan implements the payment plan engine: the `fee_plan` table and entity, CRUD with special rules (edit amount only, pause/restore), a daily scheduled task that generates recurring annual fees from patent `authorizationDate`, patent invalidation auto-pause via Spring events, and the frontend FeePlan view.

**Purpose:** Enable users to manage fee plans — create one-time fees, view automatically generated recurring plans, edit amounts, and pause plans when needed. The daily scheduled task ensures no annual fees are missed.

**Outputs:**
- FeePlan entity, FeePlanStatusEnum, DTO/VO, Mapper, Service, Controller
- V12 migration for fee_plan table
- FeePlanGenerationTask (daily 3 AM, Redis lock + atomic insert-if-not-exists)
- PatentInvalidationListener (auto-pause on invalidation event)
- FeePlan.vue with paginated plan list, edit dialog, pause/restore, create
- FundSourceSelect.vue reusable component
- feePlan.ts API service
</objective>
