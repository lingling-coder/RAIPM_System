<objective>
**Plan 02-05: Multi-Dimensional Fee Statistics Dashboard with Excel Export**

This plan implements the fee statistics backend (aggregation queries) and frontend (overview cards + cross-filter table + Excel export). All statistics are computed via MySQL GROUP BY aggregation with CASE WHEN, using a dedicated FeeStatsMapper.xml for custom SQL. Statistics are co-located in the achievement-fee module per D-30.

**Purpose:** Give management visibility into fee expenditures across 4 dimensions (department, year, patent type, funding source) with the ability to export for reporting.

**Outputs:**
- FeeStatsVO for structured stats response
- FeeStatsMapper.xml with custom aggregation SQL (GROUP BY + CASE WHEN)
- FeeStatsService with overview and dimension-groups queries
- FeeStatsController at /api/fees/stats (overview + dimension + export)
- V15 migration for composite stats index on fee_record
- FeeStats.vue with 4 overview cards + cross-filter el-table + Excel export button
- feeStats.ts frontend API service
</objective>

Aggregation SQL pattern (from RESEARCH.md):
```xml
<select id="summarizeByDimension" resultType="FeeStatsVO">
    SELECT
        COALESCE(SUM(CASE WHEN fr.status = 'paid' THEN fr.paid_amount ELSE 0 END), 0) AS totalPaid,
        COALESCE(SUM(CASE WHEN fr.status = 'pending' THEN fr.amount ELSE 0 END), 0) AS totalPending,
        COALESCE(SUM(CASE WHEN fr.due_date < CURDATE() AND fr.status = 'pending' 
                     THEN fr.amount ELSE 0 END), 0) AS totalOverdue,
        COALESCE(SUM(fr.amount), 0) AS totalAmount,
        COUNT(*) AS recordCount,
        fr.${dimension} AS dimensionValue
    FROM fee_record fr
    <where>...</where>
    GROUP BY fr.${dimension}
    ORDER BY dimensionValue
</select>
```

Composite index for stats performance:
```sql
CREATE INDEX idx_fee_record_stats ON fee_record (status, due_date, funding_source, dept_id);
```

         ```sql
         SELECT

           COALESCE(SUM(CASE WHEN fr.status = 'paid' THEN fr.paid_amount ELSE 0 END), 0) AS totalPaid,
           COALESCE(SUM(CASE WHEN fr.status = 'pending' THEN fr.amount ELSE 0 END), 0) AS totalPending,
         ```

         - Similar to getOverview but with `GROUP BY` and `AS dimensionValue`

         - For dimension = 'funding_source': return the funding_source code, GROUP BY fr.funding_source
       - statsWhereClause SQL fragment:

         ```sql
         <where>
           <if test="deptId != null">AND fr.dept_id = #{deptId}</if>
           <if test="year != null">AND YEAR(fr.due_date) = #{year}</if>
           <if test="patentType != null">AND p.patent_type = #{patentType}</if>
           <if test="fundingSource != null">AND fr.funding_source = #{fundingSource}</if>
         </where>
         ```

       - All fee_type and funding_source codes are stored as-is; frontend maps to labels

Throws IllegalArgumentException on invalid value |
