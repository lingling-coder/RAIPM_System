<objective>
**Plan 02-03: 4-Tier Alert Engine with Notification Center Integration**

This plan implements the alert engine: alert_record table, AlertLevelEnum with 4 tiers (BLUE/YELLOW/ORANGE/RED), daily scan task at 4 AM that classifies pending fee records by due date proximity, persists alert records with deduplication, and sends in-app notifications. Extends NotificationCenter.vue with a new ALERT tab. Adds color-coded alert tags to the fee ledger list.

**Purpose:** Automatically detect and notify users about upcoming and overdue fee deadlines, preventing missed payments. All alerts have full history in alert_record table.

**Outputs:**
- AlertRecord entity and AlertLevelEnum (BLUE=30d, YELLOW=15d, ORANGE=7d, RED=overdue)
- V13 migration for alert_record table (separate from notification per D-20)
- AlertScanTask (daily 4 AM, Redis lock + idempotency)
- Alert dedup via INSERT WHERE NOT EXISTS or Java-level check
- AlertRecordController for querying alert records
- NotificationCenter.vue extended with "费用预警" tab (WarningFilled icon, distinct color)
- Alert level color tags on fee ledger rows
- alertRecord.ts frontend API service
</objective>
