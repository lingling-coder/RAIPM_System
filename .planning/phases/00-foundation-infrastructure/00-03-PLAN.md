    Create the full JWT authentication backend stack.

    1. Create `JwtTokenProvider.java` in `com.institute.achievement.framework.security`:
       - Uses io.jsonwebtoken (jjwt 0.12.5): Jwts.builder(), Jwts.parser().verifyWith()
       - Keys: generate HMAC-SHA256 key from a configurable secret string (from application.yml: `jwt.secret`, default "AchievementJWTSecretKeyForHMACSHA256LengthRequirement2024") stored in config
       - `generateAccessToken(Long userId, List&lt;String&gt; roles, Long deptId, List&lt;String&gt; permissions)`:
         - Claims: sub=userId.toString(), roles (List&lt;String&gt;), deptId (Long), permissions (List&lt;String&gt;)
         - Issued at: now, expiration: now + 2 hours
         - Sign with HMAC-SHA256
       - `generateRefreshToken(Long userId)`:
         - Claims: sub=userId.toString(), type="refresh"
         - Expiration: now + 7 days
       - `validateToken(String token)`: parse, verify signature, check expiry, return true/false
       - `getUserIdFromToken(String token)`: extract subject as Long
       - `getRolesFromToken(String token)`: extract roles claim as List&lt;String&gt;
       - `getDeptIdFromToken(String token)`: extract deptId claim
       - `getPermissionsFromToken(String token)`: extract permissions claim
       - `isRefreshToken(String token)`: check type claim equals "refresh"
       - `tokenBlacklist`: use Redis Set for blacklisted tokens (logout)
       - `blacklistToken(String token, Duration duration)`: add to Redis with TTL matching token remaining life
       - Annotate class with @Component, read jwt.secret from @Value

    2. Create `UserDetailsServiceImpl.java`:
       - Implements UserDetailsService
       - `loadUserByUsername(String username)`:
         - Query SysUser by username from database (via UserMapper or injected repository)
         - Check: if user not found -> throw UsernameNotFoundException
         - Check: if deleted=1 -> throw DisabledException("账户已删除")
         - Check: if status=0 -> throw DisabledException("账户已停用")
         - Check: if lockout_until is not null and in future -> throw LockedException with remaining minutes
         - Return Spring Security User with userId stored as principal (custom UserDetails implementation)
       - Create inner `JwtUser` class implementing UserDetails with: userId, username, password, roles (Collection&lt;GrantedAuthority&gt;), deptId, permissions
       - Load roles from sys_user_role JOIN sys_role WHERE user_id matches

    3. Create `AccountLockoutService.java`:
       - `recordLoginFailure(String username)`:
         - Increment login_failures in sys_user
         - If login_failures >= 5: set lockout_until = now + 30 minutes, reset login_failures = 0
       - `resetLoginAttempts(String username)`:
         - Reset login_failures = 0, clear lockout_until
       - `isAccountLocked(String username)`:
         - Query lockout_until, return true if now &lt; lockout_until
       - Use SysUserMapper directly (injected)

    4. Create `TokenRefreshService.java`:
       - `refreshAccessToken(String refreshToken)`:
         - Validate refreshToken via JwtTokenProvider.validateToken and isRefreshToken
         - Extract userId from token
         - Fetch user from DB, re-check status/deleted/lockout
         - Load current roles + permissions
         - Generate and return new AccessToken
       - `removeRefreshToken(String refreshToken)`: blacklist the old refresh token

    5. Create `SecurityConfig.java`:
       - @Configuration, @EnableWebSecurity, @EnableMethodSecurity
       - `securityFilterChain(HttpSecurity http)`:
         - CSRF: disable (stateless JWT, D-23)
         - Session: sessionCreationPolicy = STATELESS
         - CORS: allow localhost:5173 (Vite dev), allow credentials
         - Permit-all: POST /api/auth/login, POST /api/auth/refresh, GET /api/auth/public-key (if needed)
         - Authenticate all other requests
         - Exception handling:
           - 401: return JSON {"code":401,"message":"未认证"}
           - 403: return JSON {"code":403,"message":"无权限访问"}
         - Add JwtAuthenticationFilter before UsernamePasswordAuthenticationFilter
       - `passwordEncoder()` -> BCryptPasswordEncoder (strength 10)
       - `authenticationManager()` -> configure AuthenticationManager

    6. Create `JwtAuthenticationFilter.java`:
       - Extends OncePerRequestFilter
       - `doFilterInternal(request, response, chain)`:
         - Skip filter if path matches `/api/auth/login` or `/api/auth/refresh`
         - Extract token from Authorization header: "Bearer " prefix
         - If token present:
           - Call JwtTokenProvider.validateToken
           - If invalid: set 401 response, return
           - Extract userId, roles, deptId, permissions from token
           - Create JwtUser (from UserDetailsServiceImpl inner class) and set UsernamePasswordAuthenticationToken in SecurityContextHolder
         - Call chain.doFilter()

    7. Create DTOs:
       - `LoginDTO`: username (NotBlank), password (NotBlank), remember (Boolean, optional)
       - `LoginResultVO`: accessToken, tokenType ("Bearer"), expiresIn (7200), userInfo (UserVO subset: id, username, realName, deptId, deptName, roles, permissions)

    8. Create `AuthController.java` at `/api/auth/`:
       - `@PostMapping("/login")`:
         - Validate LoginDTO
         - Check account lockout via AccountLockoutService
         - Authenticate via AuthenticationManager.authenticate()
         - On failure: record login failure, throw BadCredentialsException or LockedException
         - On success: reset login attempts, generate accessToken and refreshToken
         - Set refreshToken as httpOnly cookie (path="/api/auth/refresh", maxAge=7d, secure=false for dev, httpOnly=true, sameSite=Lax)
         - Return LoginResultVO with accessToken
       - `@PostMapping("/refresh")`:
         - Read refreshToken from cookie "refreshToken"
         - Call TokenRefreshService.refreshAccessToken
         - Return new accessToken
       - `@PostMapping("/logout")`:
         - Get current user's accessToken from header
         - Blacklist both access and refresh tokens
         - Clear refreshToken cookie
         - Return success

         refresh-token-expiration: 604800000  # 7 days in ms
