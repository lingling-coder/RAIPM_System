## Spacing Scale

Element Plus uses its own CSS variable spacing system. Document the values used across Phase 0 layouts:

| Token | Value | Element Plus Map | Usage |
|-------|-------|-----------------|-------|
| 2xs | 4px | -- | Icon-to-text gap in ElTag, compact badge spacing |
| xs | 8px | --el-table-cell-padding (vertical) | Table cell padding, button group gap, small form spacing |
| sm | 12px | --el-component-size-small gap | Form item label-to-input gap, compact card inner padding |
| md | 16px | --el-component-size gap (default) | Page content padding (left/right), card body padding, drawer body padding |
| lg | 20px | --el-card-padding | Card inner padding, dialog body padding, section inner spacing |
| xl | 24px | -- | Section heading to content gap, table toolbar top margin, form row bottom margin |
| 2xl | 32px | -- | Page title to content gap, major section separator, drawer footer padding |
| 3xl | 48px | --el-component-size-large gap | Page-level padding top/bottom, modal to window edge |

**Sidebar:** Vue Pure Admin default sidebar width = 220px (expanded), 64px (collapsed).
**Navbar:** Top navbar height = 48px. Breadcrumb + tabs bar height = 40px.
**Page padding:** 16px left + right, 16px top + bottom (consistent across all management pages).

Exceptions: None for Phase 0. All spacing conforms to the Element Plus 4/8/12/16/20/24/32/48 scale derived from the component system.

## Typography

Element Plus default font sizes, applied globally. Chinese enterprise management system standard sizing.

| Role | Size | Weight | Line Height | Element Plus Token | Usage |
|------|------|--------|-------------|--------------------|-------|
| Heading | 18px | 600 (bold) | 1.4 | --el-font-size-large | Page titles (页面标题), section headings (卡片标题, 抽屉标题) |
| Subheading | 16px | 600 (bold) | 1.4 | --el-font-size-medium | Sub-section headings, drawer sub-titles |
| Body | 14px | 400 (regular) | 1.5 | --el-font-size-base | Default body text, table cell content, form labels, paragraph text |
| Small | 12px | 400 (regular) | 1.5 | --el-font-size-small | Auxiliary text, ElTag text, helper text, table footer, pagination info |
| Mini | 12px | 400 (regular) | 1.5 | --el-font-size-extra-small | Badge text, timeline time text (rare use) |

**Usage rules:**
- Page titles use 18px heading.
- Table column headers use 14px weight 600 (slightly bolder than cells).
- Form labels use 14px weight 400.
- All ElTag text uses 12px weight 400.
- No custom font weights beyond 400 and 600. Never use 300, 500, or 700.

### Accent Reserved For (10%)

The primary color #409EFF is reserved exclusively for:
- Primary action buttons (确定, 保存, 提交, 搜索)
- Active menu item indicator (sidebar highlight)
- Input field focus border
- Link text (查看详情, 更多)
- Selected state in ElTable (checkbox selected)
- ElTag--primary type tags
- ElTree node selected state
- ElPagination current page number

**Do NOT use primary color for:**
- Non-interactive decorative elements
- Background fills (unless hover/active state)
- Disabled states (use --el-text-color-placeholder)
- Error or warning states (use danger/warning colors)

### Accent Reserved For (10%)

The primary color #409EFF must NOT be scattered arbitrarily. Specific reservation list (PASS/FAIL check):

- Primary action buttons: YES
- Active/focus input borders: YES
- Navigation active indicator: YES
- Link text: YES
- Table row selected indicator: YES
- Loading spinners: YES
- Anything decorative: NO
- Non-interactive headings: NO
- Tag backgrounds other than ElTag--primary: NO
- Icon-only decorative icons: NO
- Disabled elements: NO (use #C0C4CC)

## Page-Level Contracts

### P-01: Login Page

**Source:** CONTEXT.md D-38 (minimal style, no brand info)

**Layout:**
- Full-screen centered card (min-width 400px, max-width 440px)
- No header, no sidebar, no navigation
- No brand logo or system name (D-38 explicitly says "极简风格，仅输入框+登录按钮，无品牌信息展示")
- Background: solid #F5F7FA (no gradient, no imagery)

**Element Plus Components:**
- `ElForm` (username + password fields, validation rules)
- `ElInput` for username (prepend icon: User)
- `ElInput` for password (type="password", show-password toggle, prepend icon: Lock)
- `ElButton` type="primary" full-width for login
- `ElAlert` for error messages (type="error", show only on failure)
- `ElCheckbox` for "记住我" (optional, remember username)

**Key Interaction Patterns:**
- Form validates on submit (username required, password min 8 chars)
- Submit button shows `v-loading` during authentication
- On 401: show error alert "用户名或密码错误"
- On locked: show error alert "账户已锁定，请30分钟后再试"
- On success: redirect to welcome/dashboard page via router.push
- Browser auto-fill allowed (autocomplete="username" / "current-password")
- Enter key submits form

**States:**
- Initial: empty inputs, cursor in username field
- Filled: inputs populated, submit button enabled
- Loading: submit button loading, inputs disabled
- Error: ElAlert with error message above form
- Locked: ElAlert with lock message, "联系系统管理员" link

- On create: password field (ElInput, min 8, auto-generated with option to customize)
- On edit: no password field -- password reset is separate action
- Footer: "取消" + "保存" buttons

- File upload via ElUpload (drag-and-drop, accept .csv)
- Template download link: "下载导入模板"
- Strategy: 覆盖更新 (matching by employee ID)
- Preview table: show first 5 rows after upload
- Import button: primary, triggers batch import
- Progress bar during import (ElProgress)
- Result summary: "成功导入 X 条，更新 Y 条，失败 Z 条"

**Pagination:**
- `ElPagination` at table bottom
- Default: 20 per page
- Options: 10 / 20 / 50 / 100
- Layout: "total, sizes, prev, pager, next, jumper"

**Key Interaction Patterns:**
- Search triggers immediate query (no debounce needed for button click)
- Reset clears all search fields and re-fetches
- Drawer open/close with smooth animation
- Enable/disable via ElSwitch inside drawer, or inline button with confirmation (D-19)
- Delete = soft delete (D-20), confirmation dialog "确认删除" with reason
- Password reset triggers confirmation dialog, then success message
- Batch delete: select rows, click "批量删除", confirm dialog shows count

**States:**
- Empty: empty state component (see copywriting contract)
- Loading: ElTable v-loading directive
- Error: ElAlert type="error" above table, "数据加载失败，点击重试" link
- Drawer loading: ElDrawer shows v-loading during save API call

## Interaction Patterns (Cross-Cutting)

### IP-01: Status Display via ElTag

All status fields across Phase 0 pages use `ElTag` with consistent color mapping:

| Status | ElTag Type | Example |
|--------|-----------|---------|
| 正常 / 启用 / 成功 | success (#67C23A) | User status "正常" |
| 停用 / 禁用 / 失败 | danger (#F56C6C) | User status "停用" |
| 待审核 / 待确认 | warning (#E6A23C) | Pending items |
| 已归档 / 已注销 | info (#909399) | Archived records |
| 进行中 / 生效中 | primary (#409EFF) | Active processes |

**Usage:**
- All status ElTags use `size="small"` (except table headers)
- Tags in table cells: `effect="plain"` (light background)
- Tags in filters/drawers: `effect="light"` (solid background)

### IP-02: Confirmation Dialogs for Destructive Actions

All destructive actions use `ElMessageBox.confirm`:

```javascript
ElMessageBox.confirm(
  '确定要删除用户"张三"吗？此操作将逻辑删除该用户，历史数据保留。',
  '确认删除',
  {
    confirmButtonText: '确定删除',
    cancelButtonText: '取消',
    type: 'warning',
    draggable: true,
  }
)
```

Rules:
- Title is always the action name (e.g., "确认删除", "确认停用"), never a question
- Message includes the specific item name and consequence
- Confirm button text = specific action verb (e.g., "确定删除"), never generic
- `type: 'warning'` for all destructive operations
- `type: 'info'` for non-destructive confirmations (启用, 重置密码)

### IP-03: Drawer-Based Editing (Not Dialog)

Phase 0 uses `ElDrawer` (slide from right) for all CRUD editing forms, never `ElDialog` for editing (D-06).

| Page | Drawer Width | Content |
|------|-------------|---------|
| User Management | 500px | ElForm for user fields + role multi-select |
| Role Management | 600px | ElTree for permission assignment |
| Department Management | 400px | ElForm for department fields |
| Data Dictionary (entries) | 500px | ElForm for dict entry fields |
| Data Dictionary (categories) | N/A | Uses small ElDialog (not drawer -- category name is minimal) |
| API Integration Config | 600px | ElTabs with 3 tab panels |
| Audit Log Detail | 600px | ElDescriptions (read-only) |
| Profile Center (edit info) | 450px | ElForm for name/phone/email |

Drawer rules:
- `direction="rtl"` (right to left)
- `size` as specified above (width in px)
- `close-on-click-modal="true"`
- Footer always has "取消" + "保存" buttons
- Save button shows v-loading during API call
- On successful save: ElMessage.success, close drawer, refresh table

### IP-04: Batch Operation Support

Batch operations use ElTable selection (checkbox column):

| Page | Batch Action | Behavior |
|------|-------------|----------|
| User Management | 批量删除 | Select users, confirm, soft delete all |
| User Management | 批量启用/停用 | Select users, confirm, toggle status |
| API Config | 批量启用/停用 | Select configs, toggle status |

Pattern:
1. Select rows via checkbox column
2. Action button becomes enabled (count badge: "已选 X 项")
3. Click action -> confirmation dialog showing count
4. On confirm: loop API calls with progress indicator
5. Result summary: "成功: X, 失败: Y"

### IP-05: Search + Reset Form Patterns

All management pages with search use the same pattern:

```html
<el-form :model="searchForm" inline>
  <el-form-item label="关键词">
    <el-input v-model="searchForm.keyword" placeholder="搜索..." />
  </el-form-item>
  <el-form-item label="状态">
    <el-select v-model="searchForm.status" placeholder="全部" clearable />
  </el-form-item>
  <el-form-item>
    <el-button type="primary" @click="handleSearch">搜索</el-button>
    <el-button @click="handleReset">重置</el-button>
  </el-form-item>
</el-form>
```

Rules:
- Search button: type="primary" (only primary action in the toolbar)
- Reset button: type="default" (non-primary)
- Reset clears all fields AND re-fetches data
- Search results update ElTable + ElPagination
- "搜索" is always the leftmost button, "重置" to its right

### IP-06: Empty/Loading/Error State Handling

Global pattern for all data-fetching pages:

**Empty State:**
- ElEmpty component (image="empty", default Element Plus illustration)
- Description text: specific to each page (see Copywriting Contract)
- Action button: if applicable, "新增" button below empty state

**Loading State:**
- ElTable: `v-loading="tableLoading"` directive
- ElDrawer: loading on save button during API call
- Initial page load: full-content ElSkeleton (if page load > 300ms)
- Search/re-fetch: table-level loading only (no skeleton)

**Error State:**
- ElAlert type="error" above table: "数据加载失败"
- "点击重试" link inside the alert
- On retry: re-triggers API fetch
- Network timeout: specific error message, not generic

### IP-07: Pagination Pattern

All table pages use identical pagination:

```html
<el-pagination
  v-model:current-page="pagination.page"
  v-model:page-size="pagination.pageSize"
  :page-sizes="[10, 20, 50, 100]"
  :total="pagination.total"
  layout="total, sizes, prev, pager, next, jumper"
  @size-change="handleSizeChange"
  @current-change="handleCurrentChange"
/>
```

- Default page size: 20
- Total shown: "共 X 条"
- Jumper enabled (user can type page number)
- All pagination is server-side (backend handles offset/limit)
