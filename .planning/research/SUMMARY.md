# 研究综合汇总 — 科研成果与知识产权管理系统

**Generated:** 2026-06-16

## 执行摘要

这是一个替代Excel台账的科研成果与知识产权全生命周期管理系统，覆盖论文、专利、软著三大成果类型。研究结论明确：Phase 1使用Spring Boot 4.1单体应用 + Vue 3 + Element Plus + MySQL，解决三大痛点（电子台账替代、费用逾期预防、基础统计），Phase 2演进至分布式架构。

**最大风险不是技术而是用户采纳** — 如果 >80% 的字段不是自动填充的（DOI自动补全、申请号查询），研究人员会弃用系统。

**第二大风险是数据架构** — 多态关系设计错误（通用relation_type模式）将导致性能从第一天起就不可接受。

## 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 后端框架 | Spring Boot 4.1 + JDK 21 | Jakarta EE 11、虚拟线程、LTS |
| ORM | MyBatis-Plus 3.5.x | 复杂多表Join，中国企业标准 |
| 前端 | Vue 3.4 + Element Plus 2.9+ | 最成熟的企业中后台UI |
| 数据库 | MySQL 8.4 LTS + Redis 7.x | 成熟稳定，Phase 1足够 |
| 搜索 | MySQL ngram(Phase1) → ES 8+IK(Phase2) | 5万条为切换阈值 |
| 多态关联 | Exclusive Arc模式 | 避免N+1查询 |
| 权限过滤 | SQL层拦截器注入 | 避免应用层循环 |

## Phase 1核心功能（15项）

成果登记（论文/专利/软著）+ 部门审批流 + 专利费用台账 + 四级预警 + 基础统计看板 + 基础检索 + RBAC权限 + 审计日志 + 站内消息/邮件 + 人员导入 + 文件上传 + 成果注销 + 统一登录 + 每日自动备份

## Phase 1.x快速取胜

DOI自动补全（上线后最快被投诉的功能）、批量缴费单生成、自定义导出、引用批量导入、提醒模板、高级组合检索

## Phase 2延迟项

转化生命周期跟踪、CNIPA专利状态同步、引文API集成、自定义报表构建器、移动端H5、ES全文搜索、涉密权限体系、短信、财务集成、成果-课题关联

## 关键陷阱（Phase 1必须解决）

| 陷阱 | 预防策略 |
|------|---------|
| 多态关联N+1 | Exclusive Arc模式（第一天开始） |
| RBAC权限N+1 | SQL层拦截器，非应用层循环 |
| 涉密数据泄露 | 独立Schema + 加密存储 |
| 审计日志篡改 | 独立DB + 只读凭证 + 哈希链 |
| 文件权限绕过 | 代理转发 + UUID文件名 |
| 年费计算错误 | 版本化费用规则引擎 |
| 用户采纳阻力 | >80%自动填充 + 个人价值可见 |

## 建议阶段划分

1. **Phase 0 — 基础（2-3周）**: 系统管理模块（用户/角色/部门/权限），JWT认证，数据权限拦截器，审计日志表（月分区），文件存储服务
2. **Phase 1 — 核心功能（8-10周）**: 三大成果登记+审批流，费用管理+规则引擎+预警，基础看板+检索，15项标配功能
3. **Phase 1.x — 快速取胜（3-4周）**: DOI自动补全，批量缴费单，自定义导出，引用批量导入
4. **Phase 2 — 深化（8-12周）**: 转化跟踪，CNIPA同步，ES全文搜索，自定义报表，移动端，财务对接
