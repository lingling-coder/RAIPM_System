# Requirements: 科研成果与知识产权管理系统

**Defined:** 2026-06-16
**Core Value:** 确保科研成果与知识产权的全生命周期可追溯、费用不逾期、数据可分析

## v1 Requirements

### 成果登记 (REG)

- [x] **REG-01**: 科研人员可登记论文成果，填写标题、作者、期刊、DOI、ISSN/CN、卷期页码、发表年份、收录情况、影响因子、分区、摘要等字段
- [x] **REG-02**: 科研人员可登记专利成果，填写专利名称、发明人、申请号、授权号、申请日、授权日、专利类型、国别、年费下次缴费日、法律状态等字段
- [x] **REG-03**: 科研人员可登记软著成果，填写软著名称、著作权人、登记号、登记日期、版本号、软件类别等字段
- [x] **REG-04**: 系统支持DOI自动补全论文信息（对接Crossref/Scopus/OpenAlex，多源可切换优先级）
- [x] **REG-05**: 系统支持Excel/RIS批量导入已有成果数据
- [x] **REG-06**: 系统支持轻量级成果-课题关联，登记成果时可选择所属课题/项目
- [x] **REG-07**: 系统支持成果附件上传（PDF原文、证书、缴费凭证等），支持下载权限校验
- [x] **REG-08**: 系统支持成果注销/作废，留存作废原因与操作记录
- [x] **REG-09**: 同一DOI/申请号仅允许登记一条数据，重复提交自动拦截
- [x] **REG-10**: 涉密成果标记后仅授权人员可查看/下载附件

### 审批流程 (APPR)

- [x] **APPR-01**: 基础三步审批流：科研人员提交 → 部门科研秘书初审 → 系统管理员归档
- [x] **APPR-02**: 所有审批操作留存完整操作日志（操作人、时间、IP、变更前后内容）
- [x] **APPR-03**: 审批待办通过站内消息通知审批人

### 费用管理 (FEE)

- [ ] **FEE-01**: 费用台账：关联对应专利/软著，记录费用类型、金额、缴费日期、截止日期、缴费状态
- [ ] **FEE-02**: 缴费计划：支持单次缴费和周期性年费自动生成待缴计划，可手动编辑和暂停
- [ ] **FEE-03**: 四级预警：提前30天/15天/7天预警 + 逾期预警
- [ ] **FEE-04**: 二次催办：首次预警未处理，系统自动追加提醒并告知部门负责人
- [ ] **FEE-05**: 支持批量生成缴费单、批量标记缴费完成
- [ ] **FEE-06**: 按部门、年份、专利类型、经费来源多维度汇总费用支出统计

### 统计分析 (STAT)

- [ ] **STAT-01**: 标准可视化看板：年度成果数量趋势（折线图）、成果类型分布（饼图）、部门成果排行（柱状图）、专利有效/失效占比
- [ ] **STAT-02**: 所有报表支持导出Excel、PDF格式

### 全文检索 (SRCH)

- [ ] **SRCH-01**: 基础检索：按成果标题、摘要、作者/发明人、关键词检索
- [ ] **SRCH-02**: 支持中文分词、模糊匹配、关键词高亮、按相关性排序
- [ ] **SRCH-03**: 检索结果按成果类型、所属部门、年份、密级筛选
- [ ] **SRCH-04**: 检索结果自动做权限过滤，无权限数据不可见

### 申报提醒 (RMD)

- [ ] **RMD-01**: 支持多种提醒类型：项目申报、奖项申报、专利年费、软著维护、转化后评估、涉密成果定期核查
- [ ] **RMD-02**: 管理员可配置提醒事项（名称、截止日期、提前天数、责任人、紧急等级）
- [ ] **RMD-03**: 提醒模板支持预设、批量创建
- [ ] **RMD-04**: 触达方式：站内消息 + 邮件（SMTP）
- [ ] **RMD-05**: 支持回执确认，逾期未确认自动二次催办
- [ ] **RMD-06**: 紧急等级提醒支持系统全局弹窗强提醒

### 用户与权限 (SYS)

- [ ] **SYS-01**: 支持7类角色：科研人员、科研秘书、部门管理员、主管/院领导、涉密成果管理员、内审/审计人员、系统管理员
- [ ] **SYS-02**: RBAC权限模型：角色决定功能菜单和操作权限
- [ ] **SYS-03**: 部门数据隔离：SQL层自动注入部门过滤条件
- [ ] **SYS-04**: 涉密数据独立权限体系：独立Schema/实例隔离
- [ ] **SYS-05**: 支持人员/部门数据导入（CSV导入，预留HR系统/LDAP接口）
- [ ] **SYS-06**: 系统自建账号体系（JWT认证），预留LDAP/OAuth统一认证字段
- [ ] **SYS-07**: 全量审计日志：操作人、时间、IP、变更前后内容，日志不可删除篡改，按月度分区存储（留存5年）
- [ ] **SYS-08**: 数据字典管理：支持管理员在线维护字典项

### 外部API集成 (API)

- [x] **API-01**: DOI自动补全（集成Crossref/Scopus/OpenAlex，多源切换、优先级配置）
- [ ] **API-02**: 邮件推送通知（企业邮箱SMTP）
- [ ] **API-03**: 接口统一规范：配置中心管理密钥、超时时间自定义、自动重试最多3次、失败告警
- [ ] **API-04**: 接口故障时启用手工录入降级方案

### 运维与安全 (OPS)

- [ ] **OPS-01**: 每日自动全量备份数据库，备份文件留存30天
- [ ] **OPS-02**: 附件通过代理服务转发（不直接暴露存储路径），UUID文件名
- [ ] **OPS-03**: 兼容主流浏览器（Chrome、Edge、360浏览器）
- [ ] **OPS-04**: 支持50用户并发操作

## v2 Requirements (Deferred)

### 转化跟踪

- **TRNS-01**: 三种转化模式（技术转让/许可使用/作价入股）全生命周期跟踪
- **TRNS-02**: 转化节点跟踪（合同签订→收款→开票→完成），支持自定义节点
- **TRNS-03**: 合同中止/失败/作废状态处理
- **TRNS-04**: 转化后效益跟踪（3/6/12个月推送提醒填报后续效益）
- **TRNS-05**: 转化漏斗统计与可视化

### 费用管理增强

- **FEE-v2-01**: 线上缴费审批流程（缴费申请→部门审核→财务审批→支付→凭证归档）
- **FEE-v2-02**: 历史欠费追溯查询

### 统计增强

- **STAT-v2-01**: 引文分析（个人/部门/全院三级，总被引次数、H指数、篇均引用）
- **STAT-v2-02**: 自定义报表（用户自由组合统计条件与展示字段）
- **STAT-v2-03**: 定时报表推送（按月/季/年自动生成并推送站内信/邮件）

### 搜索增强

- **SRCH-v2-01**: 高级组合检索（多字段/多条件联合查询）
- **SRCH-v2-02**: 升级Elasticsearch 8.x + IK分词器 + 领域词典
- **SRCH-v2-03**: 检索日志与热门检索词统计

### 移动端

- **MOB-v2-01**: 适配手机浏览器/轻应用（H5）
- **MOB-v2-02**: 支持提醒查看、简易审批、成果查阅、消息回执

### 外部API增强

- **API-v2-01**: 专利法律状态同步（CNIPA接口，异步队列+限流）
- **API-v2-02**: 引文数据同步（Scopus/OpenAlex/CSCD）
- **API-v2-03**: 短信通知接口
- **API-v2-04**: 财务系统对接（批量对账）

### 审批增强

- **APPR-v2-01**: 可配置审批流（按成果类型/密级差异化流程，含加签/会签）

### 文件增强

- **FILE-v2-01**: 附件版本管理、在线预览、加密存储
- **FILE-v2-02**: 分布式文件存储（MinIO/SeaweedFS）

## Out of Scope

| Feature | Reason |
|---------|--------|
| 学术不端检测/查重 | 已有专用系统（CNKI/万方），系统仅留存外部检测结果附件 |
| 自动全网抓取成果 | 姓名消歧难度大，准确率仅60-70%，用户纠错时间超过手工录入 |
| 实时财务对账 | 财务系统接口不稳定，采用定期对账+人工上传凭证 |
| 职称评审自动计分 | 评分规则多变，避免系统成为"规则博弈"的政治战场 |
| AI自动撰写摘要 | 幻觉风险，科研内容准确性不可控 |
| 期刊投稿状态跟踪 | 属于稿件管理系统领域，需对接ScholarOne等不公开API |
| 开放科研数据仓库 | 数据规模和Schema完全不同，需独立基础设施 |
| 短信网关（v1） | 合规成本高，v1仅预留接口 |
| 原生移动App | H5优先，原生App视需求二期后评估 |
