科研成果与知识产权管理系统需求规格说明书

1. 项目背景与目标

1.1 项目背景

研究院日常统筹管理论文、专利、软件著作权等各类科研成果及知识产权，业务覆盖成果登记、费用缴纳、转化落地、数据统计、各类申报提醒全流程。目前主要依靠人工Excel台账管理，存在成果信息易遗漏、专利/软著缴费逾期风险高、全院成果底数不清、数据汇总耗时、跨系统数据重复录入等问题，整体管理效率偏低，无法满足数字化、规范化、智能化管控要求。

同时院内已部署人员管理、财务、文献库等业务系统，各系统数据相互独立，形成数据孤岛，亟需打通数据链路。

1.2 总体目标

搭建科研成果与知识产权一体化全生命周期管理系统，实现成果从登记、审核、缴费、维护、转化、统计到注销的闭环管理；依托自动化接口减少人工录入，配置多级预警机制规避逾期风险；提供多维度数据看板支撑管理决策；完成与院内现有业务系统对接，全面提升知识产权管理规范化、智能化水平。

1.3 细分管理目标

  1. 合规管控：落实知识产权保密要求、操作留痕、档案规范化归档，满足内控与审计要求；
  2. 风险防控：通过多级预警杜绝年费、申报事项逾期问题；
  3. 数据贯通：打通人员、财务、文献库等现有数据源，实现数据一次录入、多方复用；
  4. 分级管控：针对普通成果、涉密成果、合作成果设置差异化管理规则。

1.4 分阶段建设目标

一期（基础刚需版）

实现核心成果登记、审批、费用台账、基础预警、通用检索、基础统计看板、基础权限与核心API对接，解决人工台账、缴费逾期、基础统计三大痛点。

二期（全流程深化版）

完善成果转化全链路、费用线上审批、自定义报表、高级检索、移动端应用、第三方系统深度对接、大数据量性能优化、运维监控能力，实现全业务线上化。

1.5 数据源对接边界

对接院内统一身份/HR系统、财务系统、文献库，采用增量同步/实时调用结合模式，明确数据单向/双向流转规则，不修改原有系统底层数据。

2. 用户角色

结合研究院多层级组织架构，扩充角色体系，明确各角色职责、数据访问范围、权限隔离规则，整体采用RBAC权限模型+部门数据隔离机制。

角色 主要职责 数据访问与权限边界
科研人员 登记个人成果、维护本人成果信息、查看个人成果统计、接收各类提醒、确认提醒回执 仅查看/操作本人名下成果，可上传本人成果附件；不可查看其他人员、其他部门数据
科研秘书（部门级） 审核本部门成果、管理部门知识产权费用、跟踪成果转化、导出部门报表、发起本部门缴费/申报事项 仅管辖所属部门数据，可操作本部门全部成果、费用、转化信息；无全院数据查看权限
部门管理员 协助部门秘书管理本部门人员权限、维护部门基础数据、配合内审核查资料 部门数据只读为主，仅拥有基础配置权限，无跨部门操作权限
主管/院领导 查看全院成果统计看板、趋势分析、转化效益分析、全局数据汇总，支撑管理决策 可查看全院汇总数据、各部门排行数据；可按权限查看重点成果详情，支持数据下钻
涉密成果管理员 专项管理国防专利、涉密论文/软著，管控涉密数据访问、附件加密与流转 仅负责涉密成果全流程管理，涉密数据单独权限隔离
内审/审计人员 查阅操作日志、成果台账、费用凭证、审批记录，开展合规审计 数据只读，仅可查询归档数据与日志，无新增、修改、删除权限
系统管理员 全局权限配置、数据字典维护、提醒规则配置、外部接口配置、系统运维、日志核查、数据备份管理 拥有系统最高配置权限；可管控全系统功能、接口、安全策略，禁止随意篡改业务数据

3. 功能需求

3.1 成果登记模块

支持论文、专利、软件著作权三类成果统一登记、审核、归档，支持合作成果、涉密成果差异化管理，内置智能录入、唯一性校验、全流程审计、附件管控能力。

3.1.1 论文登记

基础字段：标题、作者列表（区分第一作者/通讯作者）、院内作者/外单位合作作者、合作单位、所属部门、期刊名称、ISSN/CN、卷期页码、发表年份、DOI、收录情况（SCI/EI/CSCD等）、影响因子、中科院分区、成果状态（在线发表/正式出版）、摘要。

扩展字段：成果密级（公开/内部/涉密）、课题依托项目。

附件：论文PDF原文、录用通知、检索证明，支持附件版本管理、在线预览、加密存储、下载权限校验。

  • 智能录入：支持通过DOI自动补全大部分字段，预留学生成果专项录入接口；
  • 唯一性校验：同一DOI仅允许登记一条数据，重复提交自动拦截并提示；
  • 涉密管控：涉密论文标记后，仅涉密管理员与授权人员可查看、下载附件。

3.1.2 专利登记

基础字段：专利名称、发明人（区分院内/外单位发明人）、专利权人、申请号、授权号、申请日、授权日、专利类型（发明/实用新型/外观设计/PCT）、国别、年费下次缴费日、年费金额、代理机构、法律状态（申请中/授权/失效/驳回）。

扩展字段：PCT国际阶段、国家阶段、国际进入日期、专利标识（普通专利/国防专利/涉密专利）、课题依托项目、经费来源。

附件：专利证书、缴费凭证、审查文件，附件支持权限管控与版本留存。

  • 状态同步：预留接口，支持通过申请号自动从官方渠道获取最新法律状态、缴费节点；
  • 状态联动：专利状态变更后，自动同步至费用管理、预警提醒模块。

3.1.3 软著登记

基础字段：软著名称、著作权人、登记号、首次发表日期、登记日期、版本号、软件类别。

扩展字段：软件功能简介、运行环境、合作单位、成果密级、依托项目。

附件：软著登记证书、源代码材料、说明文档，支持附件权限管控。

3.1.4 通用功能

1. 可配置审批流程
基础流程：科研人员提交 → 部门科研秘书初审 → 系统管理员归档；
扩展规则：支持按成果类型、成果等级、涉密属性配置差异化流程，高价值专利、涉密成果可增加院领导加签、会签环节；支持流程节点增删、审批人灵活调整。

2. 全量审计日志
对成果信息新增、修改、删除、状态变更、附件上传/下载等操作，强制记录操作人、操作时间、登录IP、变更前后完整内容，日志不可删除。

3. 成果注销/作废
支持对失效、撤回、作废成果进行状态标记，留存作废原因、操作记录，历史数据永久归档。

3.2 成果转化跟踪

覆盖技术转让、许可使用、作价入股三大转化模式，完善细分类型、异常流程、效益台账，实现转化全生命周期跟踪。

  • 转化类型细分：许可使用进一步区分：独占许可、排他许可、普通许可。
  • 核心记录字段：合同编号、交易对方、合同金额、已到账金额、转化日期、转化收益分配比例（院内/团队/个人）、收益分配台账、分配凭证、实际发放记录、依托成果编号。
  • 节点跟踪：标准节点：合同签订 → 收款 → 开票 → 完成；支持用户自定义业务节点；新增合同中止、转化失败、合同作废状态，必填异常原因、佐证材料、操作记录。
  • 效益跟踪：转化完成后，系统自动在3/6/12个月推送提醒，要求填报后续效益数据（新增产值、利润、行业影响等）；效益指标支持自定义配置。
  • 转化漏斗统计：自动统计各阶段成果数量、流转转化率，可视化展示转化整体情况。

3.3 知识产权费用管理

关联专利、软著全类费用，完善经费分类、线上审批、批量操作、多级催办，实现费用台账、缴费、审批、预警闭环。

  • 费用台账：关联对应专利/软著主体，记录：费用类型（申请费、年费、代理费、维持费、复审费等）、经费来源（院内经费/纵向课题/横向课题/外协资助）、金额、缴费日期、截止日期、凭证编号、缴费状态。
  • 缴费计划：支持单次缴费、周期性年费自动生成待缴计划，计划可手动编辑、暂停。
  • 线上缴费审批流程：缴费申请提交 → 部门审核 → 财务审批 → 费用支付 → 凭证归档，每一步留存审批意见与操作日志。
  • 预警与催办：保留四级预警：提前30天/15天/7天预警、逾期预警；补充逾期二次催办：首次预警未处理，系统自动追加提醒并告知部门负责人。
  • 批量操作与追溯：支持批量生成缴费单、批量标记缴费完成；提供历史欠费追溯查询功能。
  • 费用统计：按部门、年份、专利类型、经费来源多维度汇总费用支出。

3.4 成果统计分析

保留标准可视化看板，新增自定义报表、权限隔离、定时推送、指标维度界定，满足个性化统计与常态化报送需求。

标准看板（可视化展示）

  • 年度成果数量趋势（折线图）
  • 成果类型分布（饼图）
  • 部门成果排行（柱状图）
  • 专利有效/失效占比
  • 转化合同金额与到账金额对比
  • 转化漏斗转化率

引文分析：统计维度明确区分：个人、部门、全院三级；指标包含论文总被引次数、H指数、篇均引用（依赖外部API定期同步）。

报表能力增强

  • 支持自定义报表、多维度交叉筛选，用户可自由组合统计条件、展示字段；
  • 报表权限隔离：区分个人报表、部门报表、全院报表，按角色分配查看权限；
  • 定时报表推送：支持按月/季/年自动生成报表，通过站内信、邮件推送至指定人员。

导出功能：所有报表、看板支持导出Excel、PDF格式（含完整图表）。

3.5 申报提醒

完善提醒等级、模板、批量创建、二次催办，构建闭环提醒机制。

  • 提醒类型：项目申报、奖项申报、专利年费、软著维护、成果转化后评估、涉密成果定期核查。
  • 规则配置：管理员可配置提醒事项名称、截止日期、提前N天提醒、责任部门/责任人、提醒紧急等级（普通/重要/紧急）。
  • 触达方式：基础：站内消息 + 邮件；预留短信接口；紧急等级提醒增加系统全局弹窗强提醒。
  • 辅助功能：提醒模板、批量创建、回执确认、逾期未确认自动二次催办。

3.6 全文检索

在基础检索能力上，补充高级检索、权限过滤、检索日志，兼顾查询效率与数据安全。

  • 检索范围：成果标题、摘要、作者/发明人、关键词、申请号/专利号、合同编号等。
  • 检索能力：支持中文分词、模糊匹配、关键词高亮显示、按相关性排序；新增高级组合检索，支持多字段、多条件联合查询。
  • 结果筛选：可按成果类型、所属部门、发表/申请年份、密级、法律状态筛选结果。
  • 安全管控：检索结果自动做权限过滤，无权限人员无法查看涉密数据、其他部门/人员的隐私成果。
  • 检索运维：自动记录检索日志，统计热门检索词、高频检索行为，用于系统优化。

5. 外部API接口预留清单（关键）

接口统一遵循：后台可视化开关、在线测试、超时时间自定义；调用失败自动重试（最多3次），重试失败记录日志并向系统管理员发送告警；所有密钥、接口地址、账号等敏感参数通过配置中心/环境变量管理，禁止代码硬编码；接口异常时启用手工录入降级方案，保障业务不中断。

5.2 接口补充规范

  • 多数据源规则：配置接口调用优先级，多数据源返回数据不一致时，按预设规则做数据清洗、冲突判定，并保留原始数据日志；
  • 流量管控：针对国知局等有调用频次限制的接口，增设接口限流、异步任务队列，避免超限封禁；
  • 运维监控：搭建接口监控大盘，实时展示接口调用量、成功率、平均耗时、异常记录；
  • 数据映射：统一定义外部接口字段与系统内部字段的映射关系、数据格式转换规则。

6. 数据模型概览（关键实体）

6.1 设计原则

  1. 所有主表配套历史归档表，状态变更、数据修改后归档历史数据，提升主表查询性能；
  2. 核心表建立主键、外键、业务索引（DOI、申请号、登记号、部门ID、用户ID等）；
  3. 独立拆分附件表、数据字典表，实现通用数据统一管理；
  4. 表间关联清晰，支持联表查询、统计分析。

6.2 核心实体表清单

  • 论文表：id, title, doi, authors, outer_authors, cooperate_unit, journal, issn_cn, volume_page, publish_year, impact_factor, citation_count, included_type, partition, status, abstract, secret_level, dept_id, create_user, create_time

  • 费用表：id, relation_type, relation_id, fee_type, fund_source, amount, due_date, paid_date, voucher_no, pay_status, create_user
  • 转化项目表：id, result_type, result_id, contract_no, partner, contract_amount, received_amount, transform_type, distribute_ratio, finish_status, abnormal_reason
  • 转化收益分配表：id, transform_id, inner_ratio, team_ratio, personal_ratio, actual_amount, voucher_file, record_time
  • 提醒任务表：id, title, target_type, target_id, remind_date, remind_level, receiver_id, is_sent, is_confirm, confirm_time
  • 操作日志表：id, user_id, operate_type, table_name, record_id, old_value, new_value, ip_address, operate_time
  • 附件表：id, relation_type, relation_id, file_name, file_type, file_path, version, upload_user, secret_level, download_permission
  • 基础支撑表：用户表、部门表、数据字典表、审批流程表、接口配置表、检索日志表

6.3 关系说明

费用表、转化项目表通过relation_type+relation_id关联论文/专利/软著三大成果主体；所有业务表通过dept_id、user_id关联部门、用户表，实现数据隔离；附件表与所有成果、合同、凭证数据一对多关联。

7. 验收标准

7.1 功能验收

  1. 完整实现成果登记、审核、费用管理、转化跟踪、统计分析、申报提醒、全文检索全部功能，各类审批流程、异常状态流转正常；
  2. 附件版本管理、权限管控、审计日志、回执确认等辅助功能可用；
  3. 多级审批、差异化流程、批量操作、自定义报表等扩展功能正常生效。

7.2 性能验收

全文检索针对10000条成果数据，平均响应时间≤1秒；所有页面、接口响应时间符合非功能指标要求；50用户并发操作无卡顿、崩溃。

7.3 接口验收

  1. 至少成功对接DOI自动补全、邮件通知两大核心API，演示环境可使用Mock接口替代；
  2. 接口重试、告警、开关配置、超时设置功能正常；接口降级方案可用，接口故障时可正常手工录入数据；
  3. 提供完整接口扩展文档，明确新增API的配置、联调、运维方式。

7.4 安全与合规验收

  1. RBAC权限、部门数据隔离、涉密数据管控生效，无越权访问、越权操作问题；
  2. 操作日志完整留存，可正常查询、导出，日志无法被篡改；
  3. 数据每日自动备份功能正常，备份文件可正常恢复。

7.5 运维与兼容性验收

  1. 系统配置、字典维护、接口管理、日志查询等运维功能可用；
  2. 适配指定浏览器、操作系统及移动端核心功能；
  3. 灾备、异常告警功能正常触发。
